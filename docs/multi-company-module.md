# Module multi-entreprise

## Objectif

Permettre de vendre l'application a plusieurs compagnies sans melanger les donnees.

## Principe

Chaque compagnie doit avoir son espace separe.

## Donnees separees

- Clients
- Employes
- Sous-traitants
- Chantiers
- Devis
- Contrats
- Factures
- Catalogue
- Depenses
- Statistiques
- Reglages

## Champs techniques futurs

- tenant_id
- company_id
- created_by
- created_at
- updated_at

## Roles futurs

- Propriétaire
- Admin
- Contremaitre
- Employe
- Sous-traitant
- Comptabilite

## Prochaine etape

Ajouter la structure Supabase avec isolation par compagnie.
